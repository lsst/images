Added the ``lsst-images-admin`` command-line tool with ``convert`` (legacy FITS to new format), ``inspect`` (schema URL and format version), ``minify``, and ``extract-test-data`` subcommands.
Added ``lsst.images.serialization.backend_for_path`` and ``InputArchive.get_basic_info`` as public APIs for resolving a backend by file suffix and reading basic archive information.
Added ``lsst.images.serialization.read`` and ``lsst.images.serialization.write``, which dispatch by file suffix and infer the in-memory type from the file's schema, plus ``class_for_schema`` for looking up the registered ``ArchiveTree`` subclass for a ``schema_name``.
The ``inspect`` subcommand now also reports the registered Python class for the file's schema.
Added ``lsst.images.serialization.open``, a context-manager reader for pulling individual components (``reader.get_component("projection")``) or the whole object out of a file efficiently, layered on the new ``InputArchive.open_tree`` primitive.
``open`` and ``read`` accept an optional ``cls`` argument that validates the deserialized type and narrows the static return type.
