* Removed the per-backend ``read`` and ``read_tree`` functions (``lsst.images.fits.read``, ``lsst.images.json.read``, ``lsst.images.ndf.read``) and the ``read_fits`` / ``write_fits`` methods on ``Image``, ``Mask``, and ``MaskedImage``, along with the ``ReadResult`` return type.
  Use the generic ``lsst.images.serialization.read`` / ``write`` (with the optional ``cls`` argument) for whole-object access and ``lsst.images.serialization.open`` for components, metadata, and butler info.
  Schema-less Starlink NDFs are now read with ``lsst.images.ndf.read_starlink``.
