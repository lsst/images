Added a new experimental `lsst.images.zarr` archive backend that reads and writes Zarr v3 archives.
Supports `.zarr` and `.zarr.zip` output.
The on-disk layout is xarray/CF-shaped at the root with OME-NGFF v0.5 multiscales metadata layered on top — the same bytes are visible to xarray, GDAL, and OME-Zarr tooling.
Supports all image types.
Records the data-model schema URL (`lsst.data_model`) and the container format version (`lsst.version`) in the root group attributes, matching the schema-versioning stamps written by the FITS and NDF backends.
